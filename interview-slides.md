## Slide 4-6: 代表性工作一 — Divide-Then-Align (ACL 2025)

### 问题背景

RAG 系统面临核心矛盾：**当检索知识与模型内部知识冲突时，应该相信谁？**

现有方法的不足：
- 盲目信任检索 → 被错误信息误导
- 完全依赖参数知识 → 无法利用新信息
- 缺乏知识边界感知 → 不知何时该说"不知道"

### 核心方法：两阶段框架

```
┌─────────────────────────────────────────────────┐
│  Step 1: Divide                                  │
│  利用模型内部表征 → 识别知识边界                   │
│  划分 "模型已知" vs "模型未知"                     │
├─────────────────────────────────────────────────┤
│  Step 2: Align                                   │
│  已知 → 优先内部知识，检索作为验证                 │
│  未知 → 依赖外部检索，判断检索质量                 │
│  不可靠 → 诚实拒答                               │
└─────────────────────────────────────────────────┘
```

### 技术亮点
- 无需额外判别模型，利用模型内部表征区分知识边界
- 通过对齐训练让模型学会在适当时候说"我不知道"
- 多个 QA 基准上**同时提升准确率和校准度**

### 意义
- **对 RAG 领域**：解决了 RAG 系统可信性的核心问题
- **对工业应用**：可直接应用于任何 RAG 系统的可信性提升

## Slide 7-9: 代表性工作二 — KBQA-R1 (ICML 2026 投稿)

### 问题背景

知识库问答（KBQA）要求模型在结构化知识图谱上进行**多跳推理**。

传统 Pipeline 方法：
```
实体链接 → 关系匹配 → 查询生成 → 答案检索
         ↓ 误差逐级累积 ↓
```

### 核心方法：强化学习驱动的端到端 KBQA

| 组件 | 内容 |
|------|------|
| **端到端 RL 训练** | 不依赖 pipeline，LLM 直接学习 问题→查询→答案 |
| **多轮交互推理** | 生成查询 → 观察结果 → 修正策略 |
| **奖励设计** | 基于最终答案正确性，自主探索最优路径 |

### 核心创新
通过 RL 后训练，让 LLM 自主学会与知识库的多轮交互推理，无需人工设计中间步骤的监督信号

### 结果
- 多个 KBQA 数据集上取得 SOTA
- 模型学会自主 trial-and-error 探索
- **审稿评分：4 / 4 / 4 / 5（全正分）**

### 意义
- **对 Agent 方向**：验证了 RL 后训练可赋予 LLM 自主使用结构化工具的能力
- **可推广性**：RL 范式可迁移到 API、数据库、代码执行等更多工具调用场景
